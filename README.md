# 宇宙物理環境・計測研究室 — ローカル表示用ページ

FigmaデザインのTOPページ（`index.html`）・研究室概要ページ（`about.html`）・教授紹介ページ（`professor.html`）・学生の研究ページ（修士課程 `students-master.html` ／ 学部生 `students-undergrad.html`）を、素のHTML+CSS（レスポンシブ対応）で再現したものです。ヘッダーや各ボタン・カードから対応するページへ遷移します。

## 表示方法

`index.html` をダブルクリックしてブラウザで開くだけです。ビルドやサーバーは不要です。

## 画像について

- **アイコン（矢印・ロケットなど）** … HTMLにSVGで直接埋め込み、または `images/` のローカル画像を使用。オフラインでも表示されます。
- **写真・ロゴ** … すべて `images/` フォルダに同梱済みです。ネット接続なしで表示されます。
- 画像を差し替えたいときは、`images/` 内の**同名ファイル**を置き換えてください（用途は下表「画像を差し替える場合」を参照）。

## ファイル構成

```
lab-site/
├─ index.html              TOPページ
├─ about.html              研究室概要ページ
├─ professor.html          教授紹介ページ
├─ students-master.html    学生の研究 修士課程ページ
├─ students-undergrad.html 学生の研究 学部生ページ
├─ style.css               共通スタイルシート（全ページで共有・レスポンシブ）
├─ components.js           共通ヘッダー・フッター（全ページで共有）
├─ script.js               共通スクリプト（メニュー開閉・タブ切替・言語切替）
├─ 画像をダウンロード.command  （旧・通常は不要）画像は images/ に同梱済み
├─ images/                 画像の保存先
└─ README.md               このファイル
```

## フォントについて

見出し・本文はGoogle Fonts（Noto Sans JP）を読み込みます。オフライン時は端末標準のゴシック体で代替表示されます。

## 言語切り替え（JP/EN）について

ヘッダーの「JP/EN」をクリックすると、日本語⇔英語が切り替わります（選択はブラウザに記憶され、全ページに反映されます）。英訳は `script.js` で制御しており、追加・修正の方法は2通りです。

**方法1：本文・見出しなど個々の要素に英訳を付ける（おすすめ）**

翻訳したい要素に `data-en="英訳"` を足すだけです。日本語はそのまま中身に書きます。

```html
<p data-en="We develop next-generation observation instruments.">次世代の観測装置を研究開発しています。</p>
```

- 改行は `data-en` の中に `<br>` を書きます。
- 英文中に引用符が必要なときは、属性が壊れないようシングルクオート（`'...'`）を使ってください。
- `data-en` を付けていない要素は、英語表示でも日本語のまま残ります（英語化したくない箇所はそのままでOK）。

**ダミーテキストを実際の内容に差し替えるとき**は、その要素に `data-en="英訳"` を付け足せば、そのまま言語切り替えの対象になります（HTMLだけの編集で完結します）。

**方法2：ナビ・研究室名・フッターなど全ページ共通の固定ラベル**

`script.js` の冒頭の `DICT`（`"日本語": "英語"` の対訳表）に行を追加します。全ページに自動反映されます。

## 画像を差し替える場合

`images/` フォルダ内の**同名ファイル**を置き換えてください。各ファイルの用途は次のとおりです。

| ファイル名 | 用途 |
|---|---|
| `hero-bg.jpg` | ヒーロー背景（星空） |
| `research-master.jpg` | 学生の研究：修士課程 |
| `research-undergrad.jpg` | 学生の研究：学部生 |
| `logo-jaxa.png` | JAXAロゴ |
| `logo-perc.png` | PERCロゴ |
| `overview-hero.jpg` | 研究室概要ページのタイトル背景（天の川） |
| `planet.jpg` | 研究内容の画像（青い惑星） |
| `professor-hero.jpg` | 教授紹介ページのタイトル背景（オリオン星雲） |
| `professor.jpg` | 教授の顔写真 |
| `students-master-hero.jpg` | 学生の研究 修士課程のタイトル背景（木星） |
| `students-undergrad-hero.jpg` | 学生の研究 学部生のタイトル背景（地球） |
| `icons8-locket-60.png` | 「先頭へ戻る」ボタンのロケットアイコン |

> **タイトル背景の自動代用**：`professor-hero.jpg`（教授紹介）・`students-master-hero.jpg`（修士課程）・`students-undergrad-hero.jpg`（学部生）が見つからない場合は、`overview-hero.jpg`（天の川）が自動で代用されます。教授の顔写真 `professor.jpg` が無い場合は非表示になります。

## 備考

- お知らせ・教授挨拶などのテキストはFigmaのダミー文言をそのまま入れています。実コンテンツに差し替えてご利用ください。
- ヘッダーの「研究室概要」→ `about.html`、「教授紹介」→ `professor.html` にリンク済みです。「学生の研究」はTOPの「修士課程」「学部生」カードから、それぞれ `students-master.html` ／ `students-undergrad.html` に移動します。
- お知らせは一覧ページを作らず、TOPページ（`index.html`）の「お知らせ」セクションのみで掲載します（ヘッダーや一覧リンクは設けていません）。
- 教授の自己紹介・研究内容などのテキストはダミーです。実際の内容に差し替えてご利用ください。
- 教授紹介ページのタブ（紹介／活動実績）はクリックで内容が切り替わります（`script.js` で制御）。
- スマホ幅（画面幅900px以下）ではヘッダーがハンバーガーメニューに切り替わります。アイコンをタップで開閉でき、「学生の研究」のサブメニュー（修士課程／学部生）は展開表示されます。
- 下にスクロールすると、画面右下に「先頭へ戻る」ボタン（ロケット）が表示されます（スマホ幅のみ）。アイコンは `images/icons8-locket-60.png` を使用しています。別の画像に変えたい場合は、このファイルを差し替えるか、`script.js` 内の画像パスを変更してください。
- **ヘッダーとフッターは全ページ共通のため `components.js` に1か所だけ記述**しています。文言・リンク・ナビ項目を変えるときは `components.js` を編集すれば全ページに反映されます。各ページの HTML には `<div id="site-header"></div>` ／ `<div id="site-footer"></div>` のプレースホルダだけが入っており、読み込み時に差し込まれます（ナビの現在地ハイライトはURLから自動判定）。

## GitHubで公開・共有する

このフォルダは Git リポジトリとして初期化済みです（初回コミット済み・ブランチ `main`）。

### 方法A：コマンドで公開（おすすめ・準備済み）

1. GitHub で空のリポジトリを作成します（「Add a README」等のチェックは**外す**）。
2. このフォルダで以下を実行します（`<ユーザー名>` `<リポジトリ名>` は置き換え）。

```bash
git remote add origin https://github.com/<ユーザー名>/<リポジトリ名>.git
git push -u origin main
```

### 方法B：コマンドを使わない

GitHub でリポジトリを作成 →「Add file」→「Upload files」→ このフォルダの**中身**（`index.html` や `images` フォルダなど）をドラッグ＆ドロップ → Commit。
※ `index.html` がリポジトリの**直下**に来るようにしてください。

### メンバーへの共有（GitHub Pages で公開URLを作る）

リポジトリの **Settings → Pages → Source** を「`main` / `(root)`」にして保存すると、数分後に
`https://<ユーザー名>.github.io/<リポジトリ名>/` で実際のサイトが閲覧できます。このURLをメンバーに共有できます。

- コードを一緒に編集してもらう場合は、リポジトリの **Settings → Collaborators** でメンバーを招待してください。
- コミットの著者情報は仮設定です。変更する場合は `git config user.name "名前"` / `git config user.email "メール"` を設定し直してください。
